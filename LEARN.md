## 🧱 核心概念速成（零基础也能看懂）

### 1. Qt 框架是什么？

Qt 是一个做桌面软件的工具箱。你可以把它想象成：

```
你的代码 = 乐高搭建说明
Qt     = 乐高积木块（按钮、滑动条、窗口、画布...）
```

本项目的所有 UI 控件（按钮、滑动条、波形画布）都是 Qt 提供的。

### 2. 信号与槽（Signal & Slot）— Qt 的核心机制

**信号** = 某个事件发生了（比如"按钮被点击了"）
**槽**   = 事件发生后要执行的函数（比如"调用 load_data()"）

```python
# 连接：把"按钮点击"信号，连到"加载数据"函数
button.clicked.connect(self.load_data)
#       ↑ 信号               ↑ 槽函数
```

**类比**：门铃（信号）→ 你去开门（槽）。电线就是 `.connect()`。

### 3. memmap 是什么？

普通加载 `.mat` 会把全部数据读到内存（RAM）。2048 通道 × 30000 采样率 × 几秒 = 几百 MB。

`np.memmap` 的魔法：**数据留在硬盘上，只在你需要时才读一小块**。

```python
# 普通加载：全部进内存，吃满 RAM
data = np.load("big_file.npy")       # 500MB 全进 RAM ❌

# memmap：硬盘映射，几乎不占 RAM
data = np.memmap("big_file.raw")     # 0MB RAM ✅
# 用到时才读：data[5, 1000:2000] → 只读第5通道的1000个点
```

### 4. 对象池 (Object Pool) 是什么？

Trace 模式屏幕上只显示 6 行，但数据有 2048 个通道。**不创建 2048 条曲线，只创建 6 条，滚动时复用**。

```python
# 蠢办法：创建 2048 条曲线 = 卡死
for i in range(2048):
    curves.append(plot(...))  # 2048 条！❌

# 聪明办法：对象池 = 只创建 6 条，滚动时换数据
for i in range(6):           # 只 6 条！✅
    curves.append(plot(...))
# 用户滚到通道 100 → curves[0].setData(通道100的数据)
```

## 🔄 核心数据流：从"点击 Load Data"到"看到波形"

这是整个项目最重要的流程。搞懂了它，就搞懂了 80%。

```
用户点击 [Load Data]
        │
        ▼
┌─────────────────────────────────────────────┐
│  ① app.py: _load_signal()                   │
│     弹出文件选择框，用户选 .mat 文件          │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│  ② data.py: LoaderWorker(path).start()       │
│     在后台线程中加载文件（不卡 UI）            │
│     ┌──────────────────────────────────┐     │
│     │ 检查 .rawcache 缓存？             │     │
│     │  ├─ 有 → memmap（0 RAM，秒开）   │     │
│     │  └─ 无 → scipy/h5py 加载          │     │
│     │         → 写 .rawcache 缓存        │     │
│     │         → memmap                  │     │
│     └──────────────────────────────────┘     │
└────────────────────┬────────────────────────┘
                     │ finished 信号
                     ▼
┌─────────────────────────────────────────────┐
│  ③ app.py: on_done(data, s_freq, path)       │
│     把数据填入 SignalData 容器               │
│     sd.recon = data     ← 形状 (2048, N)     │
│     sd.s_freq = 30000   ← 采样率             │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│  ④ data.py: sd.compute_params()              │
│     取前 5 秒数据 → 每通道算 99.9 分位 →     │
│     得到 ch_amp[2048]（每个通道的幅值）       │
│     window_pts = 0.05 × 30000 = 1500 点      │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│  ⑤ grid.py: grid.build()                     │
│     创建 6 条曲线（对象池）                   │
│     创建 6 个标签                             │
│     创建 3 条斑马纹                           │
│     计算 2048 个通道的 Y 偏移 → _y_offsets   │
│     填充首屏数据                              │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│  ⑥ 用户点 [Start] → player.play()            │
│     QTimer 开始每 12.5ms 触发一次 _tick()     │
└────────────────────┬────────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────┐
│  ⑦ 每帧循环 (80fps):                        │
│     player._tick()                           │
│       → frame_ready(ptr=当前采样点)           │
│         → grid.scroll(ptr)                   │
│           → _fill_row_data()                 │
│             → data.recon[ch, ptr:ptr+1500]   │  ← memmap 零拷贝
│             → curve.setData(t, 数据片段)      │  ← 换绑数据
│         → player.ack() 释放锁                 │
└─────────────────────────────────────────────┘
```

## 📝 逐模块要点笔记

### utils.py — 工具箱

```
make_font(size)           → 创建一个 QFont 对象（字体+大小+加粗）
make_pen(color, width)    → 创建一个画笔（颜色+线宽）
format_channel_label(ch)  → 把 0 变成 "Ch1"
```

**关键知识点**：`@lru_cache` 是 Python 的缓存装饰器。同一个参数调用第二次时直接返回缓存结果，不重新创建对象。节省内存。

### config.py — 配置中心

```
配色方案（10 个 COLOR_XXX 变量）→ 被 QSS 样式表引用
QSS 样式表（STYLESHEET 字符串） → 控制所有控件的 CSS 外观
窗口启动参数（WIN_XXX）        → 控制窗口启动时的大小位置
```

**关键知识点**：QSS = Qt 版的 CSS。用 `{COLOR_BG}` 这种 f-string 语法把颜色变量插入样式表。

### data.py — 数据层（最重要！）

```
核心类:
  SignalData  — 数据容器（dataclass），存信号数据和所有显示参数
  LoaderWorker — QThread 子类，在后台线程加载文件

核心函数:
  _load_mat_sync() — 同步加载（在 LoaderWorker 线程中运行）
  _write_raw_cache() — 写 .rawcache 缓存
  _open_memmap() — 打开 memmap（零 RAM）
```

**关键知识点**：
- `np.memmap` = 硬盘上的数据直接映射成 numpy 数组
- `QThread` = Qt 的后台线程，用于耗时操作不卡 UI
- `@dataclass` = Python 的数据类，自动生成 `__init__`
- 信号跨线程：Worker 线程 emit → Qt 自动排队 → 主线程接收

### player.py — 播放引擎

```
核心类:
  Player(QObject) — 定时器驱动的播放引擎

关键设计:
  _pending 锁 → 防止上一帧没渲染完就发下一帧（丢帧保护）
  loop_mode   → True=播完从头来, False=播完自动停
  speed_mul   → 播放速度倍率
```

**关键知识点**：
- `QTimer(PreciseTimer)` = 高精度定时器，每 1000/80=12.5ms 触发一次
- `_pending` 是一个简单的互斥锁：发帧时置 True，渲染完 ack() 置 False
- `seek()` = 跳转到指定采样点，会立即 emit frame_ready 更新画面

### grid.py — 渲染核心（最复杂）

```
核心类:
  GridView(QWidget) — 双层面板（Row/Tile），对象池管理

两种模式:
  Trace (Row)  — 一行一通道，垂直堆叠，斑马纹背景
  Grid  (Tile) — TILE_COLS 列 × VISIBLE_TILE_ROWS 行的栅格

性能优化:
  对象池      → 只建 VISIBLE_ROWS 条曲线
  视口剔除    → 只填充可见通道的数据
  步进降采样  → 超 6000 点才降采样
  _t_buf 复用 → 时间轴数组只在 window_pts 改变时重建
```

**关键知识点**：
- `pg.PlotDataItem.setData()` 是增量更新，比删了重建快得多
- `pg.LinearRegionItem` = 矩形色块，用于斑马纹
- `QStackedLayout` = 卡片式布局，Trace/Grid 两个面板切换
- `eventFilter()` = Qt 事件过滤器，拦截鼠标点击判断点了哪个通道

### detail.py — 详情窗口

```
核心类:
  DetailWindow(QMainWindow) — 独立窗口，单通道放大

生命周期:
  点击通道 → new → show → 每帧 tick() → 关闭 → disconnect
```

**关键知识点**：
- `WA_DeleteOnClose` = 关闭窗口时自动 delete C++ 对象
- `WeakSet` = 弱引用集合，窗口关闭后自动移除，不阻止垃圾回收
- `player.frame_ready.connect(self._tick)` = 详情窗也监听播放信号

### app.py — 主窗口（总装）

```
核心类:
  MainWindow(QMainWindow) — 顶层窗口

职责:
  1. 组装顶栏（按钮 + 三个滑动条）
  2. 组装波形区（GridView + 通道滑块）
  3. 绑定键盘快捷键
  4. 管理 Detail 窗口（WeakSet）
  5. 连接所有信号到槽函数
```

**关键知识点**：
- `QButtonGroup` = 互斥按钮组（Trace/Grid 只能选一个）
- `sliderReleased` vs `valueChanged` = 松手时才重载 vs 拖动时即时更新
- 防抖 (debounce) = 通道滑块 30ms 内多次拖动只执行最后一次

### main.py — 入口

```
只做 6 件事：
1. 创建 QApplication
2. 设置全局字体（加粗）
3. 设置 Fusion 风格
4. 加载 QSS 样式表
5. 创建并显示 MainWindow
6. 进入事件循环 app.exec_()
```

**关键知识点**：`app.exec_()` = Qt 的事件循环。程序在这里"等待"用户操作（点击、按键等），不会执行 `exec_()` 后面的代码。

## ❓ 常见疑问

**Q: 为什么 8 个文件而不是一个？**
A: 单一职责原则。一个文件只做一件事，方便理解和修改。就像工具箱里的螺丝刀和扳手分开一样。

**Q: 为什么 Trace 模式用 PlotWidget 而 Tile 用 GraphicsLayoutWidget？**
A: Trace 只有一个图，PlotWidget 就够了。Tile 需要多个子图排成栅格，GraphicsLayoutWidget 原生支持 `addPlot(row, col)`。

**Q: 为什么 detail.py 不直接用 grid.py 的代码？**
A: Detail 显示单通道、有自己的坐标轴（µV + Time）、线更粗、窗口大小不同。复用反而更复杂。

**Q: 对象池里的 curve.setData() 是什么？**
A: 相当于"把这条曲线的数据换成新的"，而不是删除旧曲线再建新的。快很多。
